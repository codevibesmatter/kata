### Root Causes

1. **Finding**: Template says "Read the skill file at `.claude/skills/.../SKILL.md`" but agents should use the `Skill` tool
   **Broke at**: Stage 3 — template
   **Why**: The template instruction conflates "reading a file" with "activating a skill". The agent correctly chose the Skill tool, but a less experienced agent might use Read and miss the skill activation behavior.
   **Fix**: Change template instructions to "Activate the `quick-planning` skill and follow its methodology" instead of "Read the quick-planning skill file at..."

2. **Finding**: Excessive node_modules exploration during P1 (~30 tool calls before finding the answer)
   **Broke at**: Stage 3 — agent + environment
   **Why**: (a) The `find` → `fdfind` alias caused repeated failures, forcing the agent to try alternative approaches; (b) The agent didn't look for TanStack's built-in `SKILL.md` files early enough — it searched `.d.ts` type definitions instead of documentation files; (c) The Explore subagent returned late after the agent had already been searching manually.
   **Fix**: (environment) Ensure eval environment doesn't have confusing command aliases. (agent) No direct fix — the exploration eventually succeeded and the TanStack SKILL.md discovery was the breakthrough.

3. **Finding**: No enter output captured for audit
   **Broke at**: Stage 1 — harness/observability
   **Why**: The eval harness didn't capture or include the template content that `kata enter` injected into the session.
   **Fix**: Ensure the harness captures the full `kata enter` output including injected template content for audit purposes.
