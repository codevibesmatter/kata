### Root Causes

1. **Finding**: Agent wasted ~15 turns on failed bash commands during exploration
   **Broke at**: Stage 3 — agent
   **Why**: Agent used `find -type f` syntax but the environment aliases `find` to `fd` which has different flag syntax. Also attempted `Read()` on a directory and hit sibling tool call cascading errors.
   **Fix**: Agent-side — prefer `Glob` and `Bash ls` over `find` commands; avoid reading directories. No system change needed.

2. **Finding**: Enter output not captured in eval harness
   **Broke at**: Stage 1 — eval harness (not runtime)
   **Why**: The harness didn't capture the template injection/enter output, showing `[No enter output captured]`
   **Fix**: Eval harness should capture and record the full `kata enter` output including injected template content, so audits can verify what the agent was told.

3. **Finding**: Template says "Read the skill file" but agent activates skill via `Skill()` tool
   **Broke at**: Stage 3 — system (template wording)
   **Why**: The template instruction literally says "Read the quick-planning skill file at `.claude/skills/quick-planning/SKILL.md`" but the template also declares `available_skills: [quick-planning, tdd]` and the system provides a `Skill()` tool. The agent reasonably chose skill activation over file reading.
   **Fix**: Change template wording from "Read the quick-planning skill file at..." to "Activate the quick-planning skill and follow its methodology" to align with the `Skill()` tool mechanism. Or: "Use the quick-planning skill to..."
