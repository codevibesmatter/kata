### Root Causes

1. **Finding**: Agent never ran `kata can-exit`
   **Broke at**: Stage 5 — agent
   **Why**: After completing task #2 and committing, the agent went straight to a summary message. It either doesn't know about `kata can-exit` or wasn't reminded of it. The template doesn't mention `can-exit` — it only defines `global_conditions: [changes_committed]`.
   **Fix**: Template should include an explicit "## Exit" section instructing the agent to run `kata can-exit` when all tasks are complete. Alternatively, the enter output or task #2 description should mention the exit protocol.

2. **Finding**: P0 planning appears incomplete — only 2 of 5 quick-planning steps visible
   **Broke at**: Stage 3 — agent (with system contributing factor)
   **Why**: The agent did extensive codebase exploration (good) but then rushed through the actual planning artifact. The task instruction says "Apply the quick-planning methodology" with all 5 steps listed, which is clear. However, the quick-planning SKILL.md content isn't fully visible in the transcript, so we can't verify whether it reinforced the 5-step requirement strongly enough.
   **Fix**: Agent should have produced all 5 steps before marking P0 complete. The task description could add "Ensure all 5 steps are documented" as an explicit completion criterion.

3. **Finding**: Enter output not captured by harness
   **Broke at**: Stage 1 — system (harness)
   **Why**: The eval harness didn't capture the full enter output (template injection, instructions shown to agent). We can see task creation from the tool result but not the full template markdown.
   **Fix**: Harness should capture and include the complete `kata enter` output including the injected template content.

4. **Finding**: Agent wasted ~10 turns on failed shell commands
   **Broke at**: Stage 3 — agent
   **Why**: The environment aliases `find` to `fdfind` and `grep` to `rg` with incompatible flag syntax. The agent used Bash for file operations instead of the dedicated Glob/Grep/Read tools, hitting errors repeatedly before adapting.
   **Fix**: This is an agent discipline issue — the tools guidance says to prefer Glob/Grep/Read over bash find/grep. No system change needed, but a CLAUDE.md reminder about the environment's tool aliases could help.
